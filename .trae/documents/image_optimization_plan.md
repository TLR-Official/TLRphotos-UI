# 图片优化与水印功能实现方案

## 一、需求分析

### 1.1 性能问题

* 当前网站直接展示原始图片，未使用缩略图

* 原始图片体积大，导致页面加载慢

* 高额流量成本

### 1.2 安全问题

* 原始图片易被直接下载盗用

* 缺乏版权保护机制

### 1.3 解决方案需求

* 缩略图生成（建议：800x600，质量80）

* 图片压缩处理

* 自定义水印功能（拖拽定位、透明度、大小调整）

* 服务器端处理，不再保留原始图片

***

## 二、架构分析

### 2.1 当前架构问题

当前上传流程使用预签名URL直接上传到OSS，绕过了服务器：

```
前端 → 预签名URL → OSS（无服务器处理）
```

**问题**：服务器无法对图片进行压缩、缩略图生成、水印添加等处理。

### 2.2 重构后架构

```
前端 → 后端服务器（multipart上传）→ sharp处理 → OSS（缩略图+水印图）
```

***

## 三、技术方案

### 3.1 核心依赖

| 依赖     | 用途                        | 版本           |
| ------ | ------------------------- | ------------ |
| sharp  | 高性能图片处理（压缩、缩略图、水印）        | ^0.33.4      |
| multer | 处理multipart/form-data文件上传 | ^1.4.5-lts.1 |

### 3.2 缩略图规格

*不再直接使用原图！！！*

| 类型     | 尺寸            | 格式     | 质量     | 用途       |
| ------ | ------------- | ------ | ------ | -------- |
| 缩略图    | 800x800（等比缩放） | WebP   | 80%    | 列表展示+轮播图 |
| 预览图    | 原图尺寸          | WebP   | 100%   | 详情页      |
| <br /> | <br />        | <br /> | <br /> | <br />   |

### 3.3 文件命名规则

```
缩略图: photos/thumbnails/{timestamp}_{random}_thumb.webp
水印图: photos/watermarked/{timestamp}_{random}_watermarked.webp
```

## 四、实施步骤

### Phase 1：缩略图生成与压缩（后端）

#### 4.1.1 安装依赖

```bash
cd backend && npm install sharp multer @types/multer
```

#### 4.1.2 创建图片处理服务

**新建文件**: `backend/src/services/imageService.ts`

功能：

* `processImage(buffer)` - 处理上传图片，生成缩略图和预览图

* `uploadToOSS(buffer, key, contentType)` - 上传到OSS

* 错误处理机制

#### 4.1.3 修改上传路由

**修改文件**: `backend/src/routes/photos.ts`

* 添加multipart上传端点 `/upload`

* 删除或保留 `/upload/presigned`（兼容旧客户端）

* 修改 `/upload/complete` 为使用新流程

#### 4.1.4 更新数据库字段

新增字段：

* `watermarked_url` - 水印图片路径（可选）

* `watermark_config` - 水印配置JSON（可选）

### Phase 2：水印功能（后端+前端）

#### 4.2.1 后端水印处理

**修改文件**: `backend/src/services/imageService.ts`

添加方法：

* `addWatermark(buffer, config)` - 根据配置添加水印到图片

* 支持位置、大小、透明度参数

#### 4.2.2 前端水印编辑器

**修改文件**: `src/features/upload/UploadPage.tsx`

添加水印编辑模块：

* 水印预览区域（显示上传的图片）

* 可拖拽水印元素

* 透明度滑块（0-100%）

* 大小调整滑块

* 水印文字输入

#### 4.2.3 更新API接口

**修改文件**: `src/api/photos.ts`

* 修改 `completeUpload` 支持水印配置参数

## 六、前端交互设计

### 6.1 上传页面优化

1. 选择图片后显示预览
2. 水印编辑区域：

   * 显示图片预览

   * 可拖拽水印文字

4. 可选字段：水印配置

### 6.2 图片展示优化

* 列表页：显示缩略图（800px）

* 详情页：默认显示预览图（1200px），点击查看原图

* 添加图片懒加载

***

## 七、性能优化策略

### 7.1 服务器端

* 使用sharp流式处理，减少内存占用

* 设置处理超时限制（30秒）

* 添加并发处理限制

* 使用临时文件存储中间结果

### 7.2 前端

* 使用WebP格式（比JPEG小30%）

* 图片懒加载（Intersection Observer）

* 响应式图片（srcset）

* 缓存控制（Cache-Control headers）

### 7.3 OSS配置

* 设置合理的缓存策略

* 使用CDN加速

* 启用图片服务功能

***

## 八、风险与处理

| 风险                  | 处理方式                |
| ------------------- | ------------------- |
| sharp安装失败（native依赖） | 检查libvips安装，提供备用方案  |
| 大文件处理超时             | 设置合理的文件大小限制（50MB）   |
| 内存溢出                | 使用流式处理，限制并发数        |
| OSS上传失败             | 添加重试机制和错误日志         |
| 水印处理耗时              | 异步处理，先返回缩略图，后台生成水印图 |
