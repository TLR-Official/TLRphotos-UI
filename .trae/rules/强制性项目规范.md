# Trae 全局强制规则 (Global Mandatory Rules)

> ⚠️ 本文件为最高优先级指令。任何任务、任何对话轮次均必须遵守，不得因上下文切换、任务变更或用户未显式提醒而忽略。

## 1. Context 文件同步协议 (CRITICAL)

### 1.1 触发条件

**每一次**对代码库产生实际变更的操作完成后（包括但不限于：新建/修改/删除文件、调整配置、执行命令、修复Bug、重构代码），**必须立即**执行 Context 更新。

- ✅ 触发：写入代码、修改配置、执行构建/测试命令
- ❌ 不触发：纯问答、解释说明、未落地的方案讨论

### 1.2 更新格式

在 `.ai/context.md` 文件的 `## Changelog` 章节**顶部**追加条目，严格使用以下格式：

    | YYYY-MM-DD HH:mm | [类型] 简要描述（≤30字） | 关联文件/模块 |

- **时间戳**：使用 Asia/Shanghai 时区，精确到分钟
- **类型标签**：仅限 `feat` / `fix` / `refactor` / `config` / `docs` / `chore`
- **描述**：动词开头，说明"做了什么"而非"为什么做"
- **关联**：列出受影响的核心文件或模块路径

### 1.3 执行保障

- 更新 Context 是变更操作的**组成部分**，不是附加任务
- 若单次操作涉及多处变更，合并为一条 Changelog 记录
- 禁止使用"稍后更新""记得提醒我"等延迟表述
- 若 Context 文件不存在，先创建标准模板再写入

## 2. 架构红线 (Architecture Guardrails)

以下约束在任何任务中均不可违反，即使用户明确要求也需先警告风险：

- 🚫 禁止引入 SafeLine / ModSecurity / 独立 WAF 容器
- 🚫 禁止在生产服务器安装 Node.js 开发工具链或执行前端构建
- 🚫 禁止将原图存储于服务器本地磁盘（必须走 R2 签名 URL）
- 🚫 禁止开放 PocketBase Collection 的 List/View 权限给未认证用户
- ✅ 缩略图必须存为 WebP Blob 至服务器独立数据盘
- ✅ 管理后台必须经 Cloudflare Zero Trust Access 保护
- ✅ 所有环境变量通过 CI/CD 注入，禁止硬编码

## 3. 任务切换恢复协议

当检测到新任务与上一任务无关时：

1. **先读取** `.ai/context.md` 全文
2. **确认**当前架构状态与 Changelog 最新条目
3. **再开始**新任务的规划与执行
4. 禁止假设上一任务的上下文仍然有效

## 4. 规则优先级

本文件 > 用户临时指令 > AI 默认行为
若用户指令与本文件冲突，必须明确告知冲突点并获得显式确认后方可例外执行，且需在 Changelog 中标注 `[override]`。



## 2. Git 推送强制协议 (CRITICAL)

### 2.1 触发条件

**每一次**完成用户明确要求的变更 **且** Context Changelog 更新完毕后，**必须立即**执行 Git 提交与推送。

- ✅ 触发：用户指令已落地 + Changelog 已写入
- ❌ 不触发：纯讨论/方案确认/Changelog 未更新/代码存在语法错误

### 2.2 执行流程（严格按序）

1. **检查**：运行项目配置的 lint / type-check / build 命令，**任一失败则中止推送**并报告错误
2. **暂存**：仅 `git add` 本次变更相关文件，**禁止 `git add .` 或 `git add -A`**
3. **提交**：Commit Message 格式为 `[类型] 简要描述 | YYYY-MM-DD HH:mm`，与 Changelog 条目对应
4. **推送**：执行 `git push origin <当前分支>`，**禁止 `--force` / `--force-with-lease`**
5. **确认**：向用户反馈推送结果（成功/失败及原因）

### 2.3 安全红线

- 🚫 禁止推送含 `.env` / `*.key` / `pb_data/` / `node_modules/` 等敏感或生成文件
- 🚫 禁止在主分支（main/master）上直接推送未经用户确认的变更
- 🚫 禁止在推送前跳过步骤1的检查流程
- 🚫 禁止使用模糊 Commit Message（如 "update" / "fix" / "changes"）

### 2.4 异常处理

- 若推送因网络/权限失败：**重试1次**，仍失败则报告错误并保留本地提交，**不得丢弃变更**
- 若检查流程失败：**停止推送**，输出完整错误日志，等待用户指示
- 若用户要求"先别推"或"等下再推"：**立即暂停**，在 Changelog 中标注 `[push-deferred]`，待用户显式恢复后继续
